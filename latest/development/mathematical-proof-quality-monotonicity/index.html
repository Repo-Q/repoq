    Математическое обоснование монотонности качества¶

    Автор: Nikitin Kirill
    Дата: 2025-10-21
    Статус: Формальное доказательство

    0. Обозначения и модель¶

    Пусть:

    • 𝑆 — множество состояний репозитория (снимки кода)
    • Время дискретно по мерджам: \(S_0, S_1, S_2, \ldots \in S\)
    • Из каждого состояния извлекается вектор нормированных метрик \(x(S) \in [0,1]^d\):
    \[ x(S) = (c(S), h(S), \rho(S), \tau(S), o(S), \sigma(S), \ldots) \]

    где:

    • \(c\) — нормированная «средняя сложность»
    • \(h\) — нормированная «изменчивость/churn»
    • \(\rho\) — доля «горячих» файлов
    • \(\tau\) — дефицит тестов (например, \(\tau = \max(0, 0.2 - \text{tests\_ratio})/0.2\))
    • \(o\) — риск bus-factor/ownership
    • \(\sigma\) — индикатор отсутствия CI и пр.

    Интерпретация: Все компоненты трактуются как риски — чем больше значение, тем хуже.

    Частичный порядок по рискам¶

    Для любых двух состояний вводим:

    \[ S' \preceq S \iff x_i(S') \leq x_i(S) \quad \forall i \]

    Меньше — лучше.

    Интегральная метрика качества¶

    Определим \(Q: S \to \mathbb{R}\), монотонную по всем рискам:

    \[ Q(S) = Q_{\max} - \sum_{i=1}^{d} w_i \, x_i(S) - \Phi(x(S)) \]

    где:

    • \(w_i \geq 0\) — веса компонент
    • \(\Phi\) — любая покомпонентно неубывающая (монотонная) неотрицательная штрафная функция

    В практическом варианте \(Q_{\max} = 100\), а \(w_i\) и \(\Phi\) выбраны так, чтобы \(Q \in [0, 100]\).

    Hard-ограничения¶

    Задаются предикатами на векторе рисков:

    \[ C(S) \equiv \bigwedge_{j=1}^{m} g_j(x(S)) \leq 0 \]

    Примеры: - «Нет новых TODO/FIXME» ⇒ «счётчик TODO не вырос» - \(\rho(S') \leq \rho(S)\) (доля горячих файлов не увеличивается) - «Тесты зелёные»

    PCQ (ZAG)¶

    Для множества элементов поддержки \(U\) (модули или файлы):

    \[ \text{PCQ}(S) = \text{Agg}(\{u_i(S)\}_{i \in U}), \quad u_i(S) \in [0,1] \]

    где \(\text{Agg}\) — монотонный агрегатор (например, \(\min\)).

    Требование порога: \(\text{PCQ}(S) \geq \tau^*\)

    PCE (ZAG)¶

    Заявка: «Существует множество \(W \subseteq U\), \(|W| \leq k\), такое что после локальной ремедиации на \(W\) выполняется \(\text{PCQ} \geq \tau^*\)».

    Свидетель (witness): конкретный набор \(W\) (например, топ-\(k\) hotspots).

    1. Политика допуска мерджа¶

    Пусть: - \(S_t\) — текущее состояние целевой ветки (baseline) - \(S\) — кандидат (результат PR) после сборки/тестов

    Предикат допуска \(A(S_t, S)\)¶

    Мердж разрешён, если выполнены ВСЕ условия:

    1.1 Hard-гейты¶

    \(C(S) = \text{true}\) и все риски по компонентам не хуже, чем у базы:

    \[ x_i(S) \leq x_i(S_t) \quad \forall i \in H \]

    где \(H \subseteq \{1, \ldots, d\}\) — подмножество рисков, для которых требуем формальную невозрастаемость (например, TODO, hotspots-ratio, security-alerts).

    1.2 ZAG Threshold¶

    \[ \text{PCQ}(S) \geq \tau^* \]

    (с выбранным агрегатором, например \(\min\))

    1.3 Soft-цель (Quality Improvement)¶

    Скаляр качества:

    \[ Q(S) \geq Q(S_t) + \varepsilon \]

    где \(\varepsilon > 0\) — минимально значимый прирост (защитный зазор против измерительного шума).

    Политика¶

    Мерджить PR только если \(A(S_t, S) = \text{true}\)

    2. Базовые теоремы¶

    Теорема 1 (Монотонность качества)¶

    Утверждение: Если последовательность состояний \(\{S_t\}_{t \geq 0}\) строится применением только тех мерджей, для которых \(A(S_t, S_{t+1})\) истинно, и \(Q\) монотонна по каждому компоненту \(x_i\), то:

    \[ Q(S_{t+1}) \geq Q(S_t) + \varepsilon > Q(S_t) \]

    Доказательство:

    По определению допуска требуется \(Q(S_{t+1}) \geq Q(S_t) + \varepsilon\).

    Так как \(\varepsilon > 0\), получаем строгое возрастание \(Q\). □

    Следствие: \(\{Q(S_t)\}\) — строго возрастающая и, поскольку \(Q \in [0, Q_{\max}]\), ограниченная сверху последовательность; значит, число разрешённых мерджей с фиксированным \(\varepsilon\) конечно:

    \[ T \leq \left\lceil \frac{Q_{\max} - Q(S_0)}{\varepsilon} \right\rceil \]

    Теорема 2 (Сохранение минимального уровня качества по PCQ)¶

    Утверждение: Если в предикате допуска включено \(\text{PCQ}(S) \geq \tau^*\) с агрегатором \(\text{Agg} = \min\), то каждый мердж гарантирует, что худший модуль/файл (по \(u_i\)) не опускается ниже \(\tau^*\).

    Доказательство:

    При \(\min\)-агрегаторе:

    \[ \text{PCQ}(S) = \min_{i \in U} u_i(S) \]

    Условие \(\text{PCQ}(S) \geq \tau^*\) эквивалентно:

    \[ u_i(S) \geq \tau^* \quad \forall i \]

    □

    Смысл: Нельзя «компенсировать» деградацию одного узла улучшением другого — запрет на «Goodhart-подмену» по худшему элементу.

    Теорема 3 (Анти-компенсация / защита от Goodhart)¶

    Утверждение: Пусть \(Q\) монотонна по всем рискам, а hard-гейт требует по подмножеству \(H\) компонент \(x_i(S) \leq x_i(S_t)\), а также \(\text{PCQ}(S) \geq \tau^*\) с \(\min\)-агрегатором.

    Тогда никакая комбинация ухудшений по компонентам из \(H\) или по \(u_i\) не может быть «скомпенсирована» улучшениями по другим координатам так, чтобы пройти гейт.

    Доказательство:

    1. Если ухудшить любой \(x_i, i \in H\), то \(x_i(S) > x_i(S_t)\) нарушает hard-гейт
    2. Если ухудшить любой \(u_i\), то либо:
    3. \(\min_i u_i(S) < \tau^*\), что нарушает PCQ
    4. Сохраняется \(\geq \tau^*\), но тогда по определению ухудшение «худшего» узла не произошло

    □

    3. Устойчивость к шуму измерений¶

    Модель шума¶

    Пусть оценка рисков зашумлена:

    \[ \tilde{x}(S) = x(S) + \xi \]

    где по компонентам \(|\xi_i| \leq \delta_i\).

    Липшицевость Q¶

    Пусть \(Q\) липшицева по \(x\) с константой \(L_Q\):

    \[ |Q(x) - Q(y)| \leq L_Q \|x - y\|_1, \quad L_Q = \sum_i w_i + L_{\Phi} \]

    Тогда ошибка \(Q\) не более:

    \[ \Delta_Q \leq L_Q \sum_i \delta_i \]

    Лемма (Выбор \(\varepsilon\))¶

    Утверждение: Если взять \(\varepsilon > 2\Delta_Q\), то из наблюдаемого \(\tilde{Q}(S)\) и \(\tilde{Q}(S_t)\) следует истинное \(Q(S) > Q(S_t)\).

    Доказательство:

    \[ \begin{align} \tilde{Q}(S) &\geq \tilde{Q}(S_t) + \varepsilon \\ &\Rightarrow Q(S) \geq \tilde{Q}(S) - \Delta_Q \\ &\geq \tilde{Q}(S_t) + \varepsilon - \Delta_Q \\ &\geq Q(S_t) - \Delta_Q + \varepsilon - \Delta_Q \\ &> Q(S_t) \end{align} \]

    □

    Следствие: При корректной калибровке \(\varepsilon\) (или использовании доверительных интервалов/бутстреп-оценок) монотонность сохраняется в присутствии шума.

    4. Существование пути улучшений: роль PCE (witness)¶

    Постановка проблемы¶

    Пусть на шаге \(t\) выполнено \(\text{PCQ}(S_t) < \tau^*\).

    PCE-претензия¶

    PCE-претензия с witness \(W \subseteq U\), \(|W| \leq k\), утверждает, что существует локальная ремедиация на \(W\) с приращениями \(\Delta u_i \geq \gamma_i > 0\), \(i \in W\), такая что:

    \[ \min_{i \in U} u_i(S_t \oplus \Delta_W) \geq \tau^* \]

    Здесь \(S_t \oplus \Delta_W\) — состояние после локальных улучшений на \(W\).

    Конструктивность¶

    Если PR включает такую ремедиацию (или её часть, но достаточную для выполнения порога), то условие PCQ выполняется, и PR не будет блокирован этой частью гейта.

    Интерпретация: PCE даёт конструктивный план минимального «k-ремонта» и тем самым гарантирует существование пути повышения PCQ до \(\tau^*\) конечным числом PR.

    5. Оценка прироста качества от «ремонта» hotspots¶

    Модель модулей¶

    Пусть: - \(U\) — модули - Вес каждого: \(w_i^{\text{loc}} = \frac{LOC_i}{\sum_j LOC_j}\) - Локальные «утилиты»: \(u_i = 1 - \alpha \cdot \text{churn}_i^{\text{norm}} - \beta \cdot \text{hot}_i^{\text{norm}}\)

    Ремедиация¶

    При ремедиации \(i \in W\) получаем уменьшения \(\Delta \text{churn}_i, \Delta \text{hot}_i \geq 0\).

    Нижняя оценка прироста¶

    При суммирующем проектном агрегаторе:

    \[ \Delta Q \geq \sum_{i \in W} (\alpha \, \Delta \text{churn}_i + \beta \, \Delta \text{hot}_i) \cdot w_i^{\text{loc}} \geq \underline{\eta} \cdot \sum_{i \in W} w_i^{\text{loc}} \]

    где:

    \[ \underline{\eta} = \min_{i \in W} (\alpha \, \Delta \text{churn}_i + \beta \, \Delta \text{hot}_i) \]

    Следствие: Если \(W\) — топ-\(k\) по hotspot (как в witness PCE), то \(\sum_{i \in W} w_i^{\text{loc}}\) — заметная доля LOC, и \(\Delta Q\) имеет нижнюю оценку, т.е. гарантированный прирост.

    6. «Анти-регресс» по справедливости и связности (fairness-cover)¶

    Граф ко-изменений¶

    Пусть есть граф ко-изменений \(G = (V, E)\): - Модули — вершины - Веса рёбер — интенсивность co-change

    Манифест ZAG¶

    В манифесте ZAG фиксируем: - Границу \(B \subset V\) - Бюджет разреза \(C\)

    Инвариант структурной устойчивости¶

    Запрет «ухудшать» справедливость формализуется как:

    \[ \text{mincut}(G, B) \leq C \]

    Если каждый PR не увеличивает \(\text{mincut}\) сверх \(C\), то «стоимость» отделения подсистем от границы не растёт.

    Это задаёт инвариант структурной устойчивости (аналог ограничения на «хрупкость» или «монолитизацию»).

    7. Эпохи и адаптация весов (мета-оптимизация без потери монотонности)¶

    Проблема¶

    Пусть раз в спринт обновляются веса \(w_i\) (или параметры \(\Phi\)) для лучшей корреляции с реальными потерями (инциденты, MTTR).

    Вопрос: Как не сломать монотонность?

    Решение: Эпохальная стратификация¶

    1. Базовый уровень по эпохе: Фиксируем \(Q_{\text{base}}^{(e)} = Q(S_{t_e})\) на начало эпохи \(e\)

    2. Внутри эпохи: Применяем предикат допуска с фиксированными весами \(\{w_i^{(e)}\}\) и \(\varepsilon^{(e)}\) ⇒ монотонность внутри эпохи по Теореме 1

    3. На границе эпох: Обновляем веса \(\to \{w_i^{(e+1)}\}\), но также обновляем baseline:

    \[ Q_{\text{base}}^{(e+1)} := Q(S_{t_{e+1}}) \]

    Результат¶

    Получаем кусочно-монотонную траекторию, причём в каждой эпохе монотонность обеспечена строго.

    Усиленная версия¶

    Если дополнительно требовать Q-согласованность между эпохами (например, ограничения на изменение весов \(\|w^{(e+1)} - w^{(e)}\|_1 \leq \kappa\)), можно доказать глобальную квази-монотонность (отсутствие резких скачков).

    8. Исключения (waiver) без разрушения гарантии¶

    Проблема¶

    Если ввести ограниченное число «жетонов» waivers на эпоху (например, 1–2), то глобальная монотонность может нарушаться точечно.

    Решение: Балансировка долга¶

    1. Требовать баланс: За waiver-мердж следует обязательный ремедиирующий PR с \(\Delta Q \geq \varepsilon'\) (в сумме за эпоху \(\sum \Delta Q \geq 0\))

    2. Фиксация в сертификатах: В VC писать assuranceLevel="GATE:WAIVED" → долг виден и подлежит погашению

    Результат¶

    Монотонность выполняется в среднем по окну/эпохе, а локальные исключения формализованы и контролируются.

    9. Ответ на главный вопрос: «Почему это будет работать?»¶

    Собирая пункты вместе:

    Аспект Механизм обеспечения
    Монотонность Правило допуска \(Q(S) \geq Q(S_t) + \varepsilon\) (Теорема 1)
    Анти-компенсация Hard-гейты на ключевые компоненты рисков + PCQ с \(\min\) (Теорема 3)
    Существование пути PCE: топ-\(k\) hotspots (witness) → план ремедиаций → \(\text{PCQ} \geq \tau^*\) (Раздел 4)
    Устойчивость к шуму Выбор \(\varepsilon > 2\Delta_Q\) (Лемма об ошибке наблюдения)
    Структурная устойчивость Инвариант по mincut-бюджету в fairness-cover (Раздел 6)
    Мета-подстройка весов Эпохи с фиксацией baseline → кусочно-монотонная траектория (Раздел 7)
    Редкие исключения Формализованная система waiver с обязательным погашением → монотонность в окне (Раздел 8)

    Итог¶

    При соблюдении допусков и корректной калибровке порогов/шумов:

    1. Каждый принятый мердж не ухудшает качество
    2. При выборе \(\varepsilon > 0\) строго повышает агрегированное качество по \(Q\)
    3. Одновременно удерживает «худший» элемент системы выше допустимого порога \(\tau^*\) (PCQ/min)

    10. Практические параметры (рекомендации)¶

    Калибровка метрик¶

    1. Нормировка: \(x_i \in [0,1]\), проверить липшицевость \(Q\)
    2. Оценка шума: Измерить \(\Delta_Q\) эмпирически
    3. Выбор \(\varepsilon\): Взять \(\varepsilon = 0.2\)–\(0.5\) (или \(> 2\Delta_Q\))

    Настройка PCQ¶

    1. Агрегатор: Использовать \(\text{Agg} = \min\)
    2. Порог: \(\tau^* \in [0.75, 0.9]\) — гарантирует «неопускание» худшего модуля

    PCE witness¶

    1. Размер witness: \(k \in \{3, 5, 8\}\)
    2. Формирование: Топ-hotspots по масштабу (вес LOC \(\times\) hotness)

    Адаптация весов¶

    1. Частота: Обновлять \(w_i\) по эпохам, а не в каждом PR
    2. Причина: Иначе базовый критерий сравнения «плывёт»

    Обработка шума¶

    Использовать: - Повторные замеры (2–3 прогона) - Доверительные интервалы - Бутстрэп

    12. Итоговая формулировка гарантии¶

    Теорема (Монотонная мета-оптимизация качества)¶

    При выполнении условий:

    1. Все мерджи проходят предикат допуска \(A(S_t, S_{t+1})\)
    2. \(Q\) монотонна по компонентам рисков \(x_i\)
    3. \(\varepsilon > 2\Delta_Q\) (защита от шума)
    4. PCQ с \(\min\)-агрегатором и \(\tau^* \in [0.75, 0.9]\)
    5. Hard-гейты на критические риски \(H\)

    Гарантируется:

    \[ \forall t: \quad Q(S_{t+1}) > Q(S_t) \quad \land \quad \min_{i \in U} u_i(S_t) \geq \tau^* \]

    Физический смысл: Каждый принятый мердж: 1. Строго улучшает агрегированное качество \(Q\) 2. Не деградирует худший элемент системы ниже порога \(\tau^*\) 3. Не компенсирует ухудшения улучшениями (анти-Goodhart)

    Математическая «гарантия»¶

    Базируется на: - Монотонном агрегаторе качества - Жёстких некомпенсируемых ограничениях и порогах по худшему элементу (PCQ/min) - Конструктивном существовании улучшений (PCE/witness)

    При корректной калибровке \(\varepsilon\) и управлении шумом каждый принятый мердж либо строго увеличивает \(Q\), либо (в режиме \(\varepsilon = 0\)) не уменьшает его, сохраняя уровень минимального качества по компонентам.

    13. Связь с реализацией RepoQ¶

    Текущая реализация¶

    # repoq/quality.py
Q = 100 - 20*complexity - 30*hotspots - 10*todos

    Соответствие формуле:

    \[ Q = 100 - w_c \cdot c - w_h \cdot h - w_t \cdot \tau \]

    где \(w_c = 20, w_h = 30, w_t = 10\).

    Hard constraints¶

    constraints = {
    "tests_coverage_ge_80": tests_coverage >= 0.8,
    "todos_le_100": todos_count <= 100,
    "hotspots_le_20": hotspots_count <= 20,
}

    Соответствие: Предикаты \(C(S)\) на критические риски.

    Необходимые дополнения¶

    1. PCQ/PCE интеграция (ZAG): 

      pcq = min(module_scores)  # min-aggregator
assert pcq >= 0.82  # τ* threshold

    2. Noise handling (\(\varepsilon\) calibration): 

      epsilon = max(0.2, 2 * estimated_noise)
assert Q_head >= Q_base + epsilon

    3. Witness generation (PCE): 

      witness = top_k_hotspots(k=5, by=lambda f: f.loc * f.hotness)

    Заключение¶

    Представленное математическое обоснование доказывает, что предложенная мета-оптимизирующая петля качества обеспечивает монотонное (неубывающее, а при желании — строго возрастающее) качество на уровне репозитория при каждом мердже.

    Ключевые результаты:

    1. Теорема 1: Строгая монотонность \(Q\) при \(\varepsilon > 0\)
    2. Теорема 2: Защита худшего элемента через PCQ/\(\min\)
    3. Теорема 3: Анти-компенсация (защита от Goodhart)
    4. Лемма о шуме: Устойчивость при \(\varepsilon > 2\Delta_Q\)
    5. Существование пути: PCE-witness гарантирует финитность улучшений

    Это формальная основа «самоусиливающейся» петли качества в RepoQ.
